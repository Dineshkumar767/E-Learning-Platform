What is Cross-Site Request Forgery (CSRF) in Laravel?

Introduction to CSRF

Cross-Site Request Forgery (CSRF) is a security vulnerability that occurs when a malicious website tricks a user's browser into making an unintended request to another website where the user is authenticated. Laravel provides built-in protection against CSRF attacks. Here's how it works:

CSRF Protection in Laravel

Laravel protects against CSRF attacks by generating and validating CSRF tokens. Here's an example of how to include CSRF protection in your Laravel forms:

          <form method="POST" action="/submit">
            @csrf
            
          </form>

CSRF Token Verification

In Laravel, CSRF tokens are automatically generated for each user session and included in the session data and responses. When a form is submitted, the CSRF token is compared with the one stored in the session to ensure their validity. Here's an example of CSRF token verification in Laravel:

          if ($request->session()->token() !== $request->input('_token')) {
            // Invalid CSRF token
            abort(419);
          }

CSRF Middleware

Laravel includes a CSRF middleware that automatically adds CSRF protection to all routes defined in your application. The middleware checks the validity of the CSRF token on incoming requests. Here's an example of how to add the CSRF middleware to your routes:

          Route::group(['middleware' => 'web'], function () {
            // Routes with CSRF protection
          });
